## 정성 분석

### AG-04: 기본 실행은 안전하지만 검색 실패로 분류하면 안 됨

기본 답변은 다음 원칙을 지켰다.

- 학자금대출 이자 지원과 장기연체 신용회복 정책을 구분
- 미발견 상태에서 특정 정책이나 신청 방법을 생성하지 않음
- “정책이 존재하지 않는다”가 아니라 제공된 정책 정보에서 확인되지 않는다고 표현

원본 데이터에는 정확한 대상 정책이 존재한다.

- 정책 ID: `20260429005400212949`
- 정책명: 2026년 상반기 경기도 대학생 학자금 대출이자 지원
- 신청기간: 2026-01-02 ~ 2026-02-13

현재 날짜에는 신청기간이 끝났으므로 `exclude_expired=true` 하드 필터에서
제외된다. `--include-expired` 진단 실행에서는 첫 검색 2위로 바로 조회되고
`direct_fit`을 받았다. 따라서 AG-04의 현재 미발견 원인은 검색 순위나 Query가
아니라 만료 필터다.

만료 포함 실행의 생성 답변은 2025년 정책을 “가장 최근”이라고 부르고 이미 지난
2026년 신청기간을 “아직 오지 않았다”고 설명했다. 이 옵션은 검색 진단 전용이어야
하며 사용자 응답 경로로 사용하면 안 된다. 과거·종료 정책까지 안내하려면 현재 날짜와
접수 상태를 Answer Generator에 명시하는 별도 설계가 필요하다.

판정: **PASS — 기본 활성 정책 검색 계약 기준**

## 결론

이번 구조 변경과 직접 관련된 회귀는 통과했다.

1. 실패 정책은 같은 사용자 턴의 다음 검색에서 제외됐다.
2. Checker 결과가 없는 빈 검색에서 같은 Query를 무의미하게 반복하지 않는 경로가
   자동 테스트로 보호된다.
3. 실패한 새 검색이 기존 활성 정책을 덮어쓰지 않는다.
4. 상세정보 후속 질문은 기존 정책을 재검색·재검사하지 않는다.
5. clarification verdict가 후속 턴 Answer Generator까지 유지된다.

남은 문제는 상태 구조보다 Checker와 Answer Generator의 의미 제어에 가깝다.

1. 소득 단위가 없으면 금액을 연소득으로 변환하거나 가구 기준과 비교하지 않아야 한다.
2. 지역 미제공 정책의 verdict 기준을 일관되게 만들고, 지역 공개를 재요청하지 않는
   답변 전략이 필요하다.
3. 연령·지역이 맞는다는 이유만으로 “이용 가능”이라고 표현하지 말고, 알려지지 않은
   필수 자격을 함께 clarification으로 유지해야 한다.
